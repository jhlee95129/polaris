# 폴라리스 (Polaris) — One Pager

> *"운세는 절대적인 정답을 맞히는 개념이라기보다 사용자 상황과 맥락을 이해하고 방향을 제시하는 도구"*
> — 제성원 플롯 본부장 (아주경제, 2026.04)
>
> 폴라리스는 이 명제에 동의합니다.
> 다만 그 도구가 콘텐츠 카드가 아니라, 일상의 고민을 받아주는 **사주 코치**의 형태일 때 더 잘 작동한다고 봅니다.

---

## 1. 왜 사주 도메인인가

**개인적 경험**: 살아오면서 사주를 여러 번 봤습니다. 정해진 운명을 믿어서가 아니라, 큰 틀에서 삶의 방향성을 가늠하는 데 사주가 실제로 도움이 된다고 느꼈기 때문입니다. "지금 이 시기에 왜 이런 일이 생기는지", "어떤 흐름 속에 있는지"를 사주 프레임으로 바라보면 막연한 불안이 줄고 다음 행동이 명확해지는 경험을 했습니다. 그래서 이 도메인은 저에게 단순한 과제 선택이 아니라, 직접 사용자로서 불편을 느끼고 개선하고 싶었던 영역입니다.

**시장 기회**: 한국 운세/사주 시장은 연 2조 원 규모이며, 20–30대의 사주 앱 사용률이 빠르게 증가하고 있습니다. 그러나 디지털 사주 서비스의 UX는 10년 전과 본질적으로 같습니다 — 생년월일 입력, 긴 보고서 출력, 이탈.

**AI-native 기회**: Co-Star, Replika, Character.AI가 증명했듯이 사용자들은 AI와의 지속적인 대화 관계에 기꺼이 시간과 비용을 투자합니다. 사주라는 한국 고유의 자기 이해 프레임워크를 AI 라이프 코칭 형태로 적용하면 강력한 조합이 될 수 있다고 판단했습니다. 기존 사주 서비스는 "사전 설계된 콘텐츠"를 보여주는 데 그쳤지만, LLM + 대화 메모리가 등장한 지금은 사용자의 **임의의 일상 상황**에 사주 프레임으로 실시간 응답하고, 이전 대화를 **기억**하며 후속 코칭을 이어갈 수 있습니다.

**뤼튼과의 정합성**: "Bring AGI Close to People" — 사주는 한국인이 일상에서 가장 친숙하게 접하는 동양 철학이며, AI를 통해 이를 개인화된 대화형 코칭으로 전환하는 것은 뤼튼의 미션과 높은 결을 가집니다.

## 2. 기존 서비스가 해결하지 못하는 타겟과 문제

### 타겟 페르소나

**김지은, 29세, 마케터**

- 친구 만나면 MBTI/사주 이야기를 자연스럽게 합니다
- 포스텔러/점신 결제 경험이 있지만 "한 번 보고 다시 안 들어감"
- 뤼튼 플롯의 "올해 운세" 카드는 봤는데, *"오늘 팀장이랑 또 부딪혔어"* 같은 일상 상황은 어디서 물어볼지 모릅니다
- 최근 이직 고민 + 3개월 전 이별. 진짜 친구한테 매번 같은 얘기 하기는 미안합니다
- ChatGPT는 일에만 사용. 고민 상담용 AI는 아직 없습니다

→ **Unmet need**: "내 상황을 알면서 들어주고, 사주 근거로 방향을 잡아주는 코치"

### 기존 서비스의 한계 — 4분면 분석

| 서비스 | 명식 정확도 | 임의 상황 대응 | 누적 관계 | 한계 |
|--------|-----------|------------|---------|------|
| 포스텔러/점신 | O | X | X | 카테고리 메뉴 → 긴 보고서 dump. 일상 고민 불가 |
| 뤼튼 플롯 | O | 부분적 | X | 사전 설계 콘텐츠 안에서만 가능. 단발 리포트 |
| ChatGPT 프롬프팅 | X | O | X | 명식 hallucination. 매번 생년월일 재입력 |
| 오프라인 점집 | O | O | X | 1회 대면. 비용/접근성 장벽 |
| **폴라리스** | **O** | **O** | **O** | — |

핵심은 **임의 상황 대응**과 **누적 관계** — 이 두 칸은 LLM + 대화 메모리 이후에야 채울 수 있게 된 영역입니다.

플롯은 *"사용자가 질문을 고민하는 과정 자체를 줄이고 콘텐츠 선택만으로 충분한 해석을 받을 수 있도록"* 의식적으로 설계되었습니다. 그 단순화는 강점이자, 동시에 자유 대화와 누적 관계라는 영역이 구조적으로 빠지게 됩니다. 폴라리스는 이 두 칸을 정확히 겨냥한 **보완재 포지션**입니다.

## 3. 핵심 기능과 근거

### 단일 핵심 가설

> 사람들이 사주에서 진짜 원하는 건 "정해진 운명 보고서"가 아니라,
> **내 상황을 들어주고 사주 프레임으로 해석해주는 대화 상대**다.

이 한 문장이 모든 기능 결정의 기준입니다.

### 구현된 핵심 기능

**1. 명식 1회 입력 + 누적 자유 채팅**

한 번 생년월일을 입력하면 사주팔자가 계산되고, 이후에는 자유 대화만으로 상담이 이루어집니다. 4기둥(년/월/일/시주) + 오행 분포 + 십신 + 용신 + 대운(大運)까지 KASI 기반 만세력 라이브러리로 정확히 계산합니다.

- **근거**: "매번 생년월일 재입력"이라는 마찰 제거. 대화의 연속성이 리텐션으로 연결됩니다.

**2. 매 응답마다 사주 근거 + 구체적 코칭**

모든 대화에서 AI가 단순히 공감만 하는 것이 아니라, **사주적 근거와 구체적 코칭을 함께 제시**합니다. "지금 월주의 식상 기운이 강한 시기라 표현 욕구가 높아질 수 있어" 같은 명리학적 해석을 자연스럽게 녹이고, "이번 주에 팀장에게 먼저 의견을 정리해서 공유해봐" 같은 실행 가능한 행동을 제안합니다. Claude의 `tool_use`(`saju_basis`)를 통해 매 응답마다 참조한 기둥, 핵심 오행, 명리학적 추론, 코칭을 구조화 추출하여 "상세 분석 보기" 패널로 투명하게 보여줍니다.

- **근거**: 사주 앱 사용자의 핵심 불만은 "그래서 뭘 하라는 건지 모르겠다"입니다. 해석과 행동을 매 대화마다 연결하는 것이 기존 보고서형 서비스와의 본질적 차이입니다.

**3. 5명의 코칭 캐릭터 — 같은 사주, 다른 보이스**

선비(📜), 무녀(🔮), 장군(⚔️), 선녀(🌸), 도깨비(👹) — 동일한 명리학 지식 기반 위에 5가지 상담 보이스를 제공합니다. 사용자는 자신에게 맞는 톤을 골라 상담받을 수 있으며, 세션 중에도 캐릭터를 전환할 수 있습니다. 각 캐릭터는 고유한 말투, 이모지 정책, 상담 스타일을 가집니다.

- **근거**: Character.AI가 증명했듯이, 같은 내용도 "누가 말하느냐"에 따라 사용자 경험이 완전히 달라집니다. 캐릭터 선택은 사용자에게 통제감을 주고 리텐션을 높입니다.

**4. 기억하는 폴라리스 — 세션 메모리**

멀티 세션 관리로 주제별 대화를 분리하고, 재방문 시 이전 맥락 기반 follow-up 인사를 생성합니다. *"지난번 말한 이직 건은 어떻게 됐어?"* — 1회성 리포트로는 만들 수 없는 관계를 대화 메모리로 구현합니다.

- **근거**: 신뢰는 누적에서 발생합니다.

**5. 오늘의 폴라리스 — 매일 열어보는 이유**

매일 변하는 일일 운세(오늘의 코칭, 주의사항, 행운의 숫자·색상, 에너지 키워드)를 대시보드에 표시합니다. 직장/연애/재물 등 8가지 일일 코칭 카드와 추천 고민 토픽을 제공하여, 사용자가 "뭘 물어봐야 하지?"라는 고민 없이 바로 상담을 시작할 수 있습니다.

- **근거**: DAU의 핵심은 "매일 열어볼 이유"입니다. 포스텔러의 일일 운세가 가장 많이 쓰이는 기능이라는 점에서 착안했으나, 폴라리스의 일일 운세는 단발 리포트가 아니라 채팅 진입의 트리거 역할을 합니다.

**6. 대화 공유 — 바이럴 루프**

상담 대화를 공유 링크로 만들어 친구에게 보낼 수 있습니다. OG 메타 태그가 설정되어 카카오톡/SNS에서 프리뷰가 노출되며, 읽기 전용 페이지에서 사주 컨텍스트와 함께 대화 전문을 볼 수 있습니다.

- **근거**: 사주 결과를 공유하는 행위는 이미 검증된 바이럴 패턴(포스텔러, MBTI)입니다. 대화형 공유는 기존 결과 카드 공유보다 더 풍부한 컨텍스트를 전달합니다.

**7. 복채 시스템 — 지속 가능한 비즈니스 모델**

상담 1회 = 복채 1개. 가입 시 3개 무료 제공, 매일 출석 체크로 1개 충전, 상점에서 추가 구매 가능(현재 오픈 기념 무료). AI가 복채 부족을 알릴 때도 선택한 캐릭터의 보이스로 안내하여 UX의 일관성을 유지합니다.

- **근거**: 무제한 무료는 API 비용 리스크. "질문 1개 = 복채 1개"라는 직관적 단위가 가격 민감도를 낮추고, 출석 체크는 DAU를 유도합니다.

**8. RAG 기반 사주 근거 grounding**

명리학 소스 텍스트(일간별 특성, 지지 충/합/형, 격국 패턴, 대운 해석)를 벡터화하여 Supabase pgvector에 저장하고, 사용자의 일간 + 대화 주제로 2-track 검색하여 Claude 프롬프트에 주입합니다. "AI가 지어낸 해석"이 아닌 "명리학 근거에 기반한 코칭"을 보장합니다.

- **근거**: LLM이 가장 hallucinate하기 쉬운 명리학 해석을 외부 지식으로 grounding. RAG 데이터는 명리학 도서 + 위키백과 발췌 → AI 재구성 → 검수의 3단계로 생성하여 circular hallucination을 방지했습니다.

### 의도적으로 자른 것

| 자른 기능 | 이유 |
|----------|------|
| 회원가입/소셜 로그인 | 닉네임 기반 간편 접근이 가설 검증에 충분 |
| 실결제 시스템 | 복채 시스템으로 구조만 설계. 실결제는 7일 범위 밖 |
| 운세 카테고리 탭 | "자유 대화"라는 가설과 정면 충돌 |
| 화려한 명식 시각화 | 사용자 경험의 핵심이 "대화"이므로 부차적 |
| 궁합/다인 비교 | 1인 코칭이라는 핵심 가설에 집중 |

## 4. 성공 지표

### 1차 지표 — 가설 검증

| 지표 | 정의 | 목표 | 측정 의도 |
|------|------|------|---------|
| **N+1일 재방문율** | 첫 사용 다음날 재방문한 비율 | > 30% | "코치"가 되었는가? |
| **세션당 대화 턴 수** | 한 세션에서 사용자 메시지 수 | > 4턴 | 1회성 질문이 아닌 대화가 되었는가? |
| **첫 응답 이탈율** | 첫 AI 응답 후 추가 메시지 없이 이탈 | < 40% | 첫 인상이 작동하는가? |

### 2차 지표 — 제품 성장

| 지표 | 정의 | 비교 기준 |
|------|------|---------|
| **D7 리텐션** | 가입 7일 후 재방문 | 포스텔러/점신 대비 |
| **온보딩 완료율** | 명식 입력 시작 → 첫 채팅 도달 | > 70% |
| **복채 전환율** | 무료분 소진 후 구매 전환 | 가격 민감도 검증 |
| **공유율** | 대화 공유 링크 생성 비율 | 바이럴 잠재력 |
| **캐릭터 전환율** | 사용자가 캐릭터를 변경한 비율 | 캐릭터 시스템의 가치 |

### 비교 프레임

기존 사주 앱의 패턴은 **"설치 → 1회 사용 → 이탈"**입니다. 폴라리스의 핵심 KPI는 이 패턴을 깨는 것 — N+1일 재방문율이 유의미하게 높다면, "대화형 사주 코칭"이라는 가설이 작동한다는 증거입니다.
