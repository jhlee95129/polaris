# 한수 — AI 사주 라이프 코치

## 1. 도메인 선택 이유

한국 사주/운세 시장 1.4조원. MZ세대가 전문 플랫폼 사용자의 80%. AI 사주 이용률(40.7%)이 오프라인 점술(18.5%)을 이미 압도. 그러나 기존 서비스(점신, 포스텔러, 플롯)는 모두 **"해석 리포트 자판기"**에 머물러 있다. 서양에서는 Co-Star(DAU 30-40%), Replika(3,500만 유저)가 **AI 컴패니언 모델**로 거대한 성공을 거뒀지만, 한국 사주 도메인에는 아직 적용되지 않았다. 사주라는 자기 이해의 프레임워크를 AI 라이프 코치의 렌즈로 재정의한다.

## 2. 타겟과 문제

**타겟**: 2030 직장인. 사주앱을 2-3개 병행하며 "다 비슷하다"고 느끼는 사람. 사주를 보고도 "그래서 나는 뭘 해야 하지?"라는 질문이 남는 사람.

**문제 3가지**:
1. **일회성 소비** — 프로필 보고 상담 한두 번 하고 끝. 매일 돌아올 이유가 없다.
2. **해석-행동 단절** — "올해 관재수 조심" 같은 해석은 있지만, 구체적으로 뭘 하라는 건지 알 수 없다. 사용자가 원하는 건 ①감정 확인 ②행동 허가 ③구체적 방향인데, 기존 앱은 ①만 겨우 해준다.
3. **빈 textarea 공포** — ChatGPT 사주는 매번 생년월일 재입력, "뭘 물어야 하지?"에서 이탈. 자유 입력은 진입장벽이 높다.

## 3. 핵심 기능 — 두 가지 킬 기능

**F1. "오늘의 한수"** — 매일 바뀌는 맞춤 행동 카드

앱을 열면 3초 내에 오늘의 일진 + 내 원국 기반의 **구체적 행동 1가지**를 제시. "올해 금전운이 좋아요"가 아닌 "오늘 점심에 평소 말 안 하던 동료에게 먼저 커피를 제안해보세요". 매일 바뀌는 콘텐츠가 재방문 루프를 만든다. → **일회성 소비 문제 해결**

**F2. 선택형 상담 플로우** — 채팅이 아닌 구조화된 코칭 경험

빈 textarea 대신 `카테고리(4) → 세부 상황(4) → 긴급도(4) → 자유입력(선택) → 캐릭터(3)` 5단계 선택형 플로우. 탭 몇 번만으로 상담 시작. 구조화된 맥락 → Claude에 더 좋은 질문 → 더 정확한 코칭 카드(진단/한수/타이밍/피할 것/근거). → **빈 textarea 공포 해결 + 해석-행동 단절 해결**

**F3. 캐릭터 3종** — 같은 데이터, 다른 해석 렌즈

친한 언니/형(감정 프레임), 할머니 역술가(오행 프레임), 팩트체커(십신 프레임). 톤만 다른 게 아니라 해석 렌즈 자체가 다르다. "다른 관점으로 보기"로 같은 맥락을 재해석. → 세션 깊이 증가.

**F4. 사주 기반 질문 제안** — 사주가 질문을 만든다

사용자의 십신 구성, 오행 부족, 오늘의 상생상극에 기반하여 4가지 질문 칩을 자동 생성. "이 앱은 나를 안다"는 느낌 + 상담 진입 마찰 제거.

## 4. 성공 지표

| 지표 | 목표 | 측정 방법 | 실패 시 액션 |
|---|---|---|---|
| **일일 재방문율** (북극성) | ≥ 40% | "오늘의 한수" 일일 호출 수 / DAU | 한수 카드 품질 이터레이션, 푸시 알림 |
| 세션당 상담 수 | ≥ 1.5 | 상담 완료 수 / 세션 | 선택형 플로우 최적화, 카테고리 추가 |
| 상담 전환율 (홈 → 상담) | ≥ 50% | 질문 칩 탭 수 / 홈 방문 수 | 질문 제안 알고리즘 개선 |
| 캐릭터 전환 경험률 | ≥ 30% | "다른 관점" 탭 수 / 결과 표시 수 | 캐릭터 간 프레임 차이 강화 |
| 피드백 응답률 | ≥ 20% | 피드백 제출 수 / 상담 완료 수 | 피드백 타이밍/UX 변경 |

## 5. 기술 하이라이트

- **만세력 라이브러리** (KASI 기반): LLM 확률이 아닌 정확한 사주팔자 계산
- **Claude tool_use 3종**: 코칭 카드 / 프로필 요약 / 오늘의 한수 — 구조화 출력으로 UX 품질 보장
- **4계층 시스템 프롬프트**: 명리학 지식 → 오늘 천기 → 사용자 원국 → 캐릭터 톤
- **RAG**: 명리학 고전 → pgvector → 근거 기반 코칭 (프롬프트 하드코딩 대신 확장 가능)
- **선택형 플로우 빌더**: 구조화 맥락 → 자연어 질문 변환 → Claude에 전달
