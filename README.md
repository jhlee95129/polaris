# 한수 (Hansu) — AI 사주 라이프 코치

> "내 사주는 알아. 근데 **오늘 뭘 해야 하지?**"에 매일 답하는 AI 사주 코치

기존 사주 앱이 "해석 리포트 자판기"에 머물러 있는 반면, 한수는 **사주를 자기 이해의 렌즈**로 활용하여 매일 달라지는 **구체적 행동 코칭**을 제공합니다.

## 실행 방법

### 1. 환경 설정

```bash
git clone https://github.com/jhlee95129/hansu.git
cd hansu
pnpm install
```

### 2. 환경변수

`.env.local.example`을 `.env.local`로 복사하고 API 키를 입력합니다.

```bash
cp .env.local.example .env.local
```

**필수**: `ANTHROPIC_API_KEY` (Claude API)
**선택**: `OPENAI_API_KEY`, `NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SECRET_KEY` (RAG 기능)

> RAG 환경변수가 없으면 RAG 없이 동작합니다 (시스템 프롬프트 내 명리학 지식만 사용).

### 3. 실행

```bash
pnpm dev
```

`http://localhost:3000`에서 확인할 수 있습니다.

### 4. (선택) RAG 임베딩

Supabase 프로젝트에서 pgvector를 활성화하고 마이그레이션을 실행한 후:

```bash
npx tsx scripts/embed-knowledge.ts
```

## 문제 발견 → 가설 → 검증

### 발견한 문제

기존 사주 앱(점신, 포스텔러, 플롯) 사용자 여정 분석:

1. **일회성 소비**: 프로필 보고 → 상담 한두 번 하고 → 끝. 매일 돌아올 이유가 없다.
2. **빈 textarea 공포**: "뭘 물어야 하지?"에서 이탈. 자유 입력은 진입장벽이 높다.
3. **해석-행동 단절**: "올해 관재수 조심"은 알겠는데, 구체적으로 뭘 하라는 건지 모른다.

### 핵심 가설

> 사주 앱 사용자가 원하는 것은 해석이 아니라 **"오늘 내가 뭘 해야 하는지"**이다.

### 검증: 두 가지 킬 기능

**1. "오늘의 한수"** — 매일 바뀌는 맞춤 행동 카드

앱을 열면 3초 내에 오늘의 일진 + 내 원국 기반의 구체적 행동 1가지를 제시. "읽고 끝"이 아니라 "오늘 실행할 수 있는 것". 매일 바뀌는 콘텐츠가 재방문 루프를 만든다.

**2. 선택형 상담 플로우** — 채팅이 아닌 구조화된 코칭 경험

빈 textarea 대신 `카테고리(4) → 세부 상황(4) → 긴급도(4) → 자유입력(선택) → 캐릭터(3)` 5단계 선택형 플로우. 탭 몇 번만으로 상담 시작. 구조화된 맥락 수집 → Claude에 더 좋은 질문 → 더 정확한 코칭.

### 왜 채팅이 아닌 "선택형 플로우"인가

| 채팅형 | 선택형 플로우 |
|---|---|
| ChatGPT와 다를 바 없는 뻔한 UX | 기존 사주앱/AI앱에 없는 독특한 인터랙션 |
| "뭘 물어야 하지?"에서 이탈 | 탭만 하면 됨, 진입장벽 제거 |
| 맥락 수집이 사용자 역량에 의존 | 구조화된 4단계가 풍부한 맥락을 자동 수집 |
| 질문 품질 편차 큼 | 일관된 질문 품질 → 일관된 코칭 품질 |

## 핵심 UX 결정

### 사용자 여정 설계

```
[첫 방문]
  온보딩 (생년월일 → 시간 → 성별) → /home

[매일]
  앱 열기 → "오늘의 한수" 즉시 표시
       → 관심 생기면: 질문 칩 탭 → /ask 상담
       → 더 알고 싶으면: /my (내 사주 + 기록)

[재방문 루프]
  매일 바뀌는 한수 → 실행 → 피드백 → 다시 방문
```

### 3탭 네비게이션

| 탭 | 역할 |
|---|---|
| **오늘** (`/home`) | 오늘의 한수 카드 + 질문 제안 칩 + 최근 상담 |
| **상담** (`/ask`) | 선택형 5단계 상담 플로우 + 코칭 카드 결과 |
| **나** (`/my`) | 사주 프로필 + AI 분석 + 상담 기록 타임라인 |

### 의도적으로 하지 않은 것

- **계정/로그인 미도입**: MVP 마찰 제거. localStorage로 충분.
- **자유형 채팅 미도입**: 구조화된 플로우가 행동 유도에 더 적합.
- **플롯식 카테고리 큐레이션 미적용**: "오늘의 행동 코칭"이라는 고유 컨셉 유지.
- **채팅 히스토리 미도입**: 한 번의 질문 → 한 장의 코칭 카드. 세션이 아닌 카드 단위 인터랙션.

## 기술 스택

| 영역 | 기술 | 선택 이유 |
|---|---|---|
| 프레임워크 | Next.js 16 (App Router) + TypeScript + React 19 | SSR/SSG 최적화, 평가자 실행 마찰 최소화 |
| UI | Tailwind CSS v4 + shadcn/ui (OKLCH) | 일관된 디자인 시스템, 접근성 내장 |
| 사주 계산 | `@fullstackfamily/manseryeok` | KASI 기반 한국 특화 만세력. LLM 확률 대신 정확한 라이브러리 |
| AI | Claude Sonnet 4 (`@anthropic-ai/sdk`) | tool_use로 코칭 카드/프로필/일일 행동 구조화 출력 강제 |
| RAG | Supabase pgvector + OpenAI Embeddings | 명리학 고전 → 벡터 검색 → 프롬프트 주입 |
| 데이터 | localStorage (MVP) | 점진적 마이그레이션 가능한 설계 |

### 핵심 기술 결정

**1. tool_use로 구조화 출력 강제**

코칭 카드 5필드(진단/한수/타이밍/피할 것/명리학 근거), 프로필 요약 5필드, 오늘의 한수 5필드를 모두 Claude tool_use로 강제. JSON 파싱 에러 0%, UX 품질 일정.

**2. 4계층 시스템 프롬프트**

```
Layer 0: 명리학 기본 지식 (천간·지지·오행·십신 해석 원칙)
Layer 1: 오늘의 천기 (일진 + 원국과의 상생상극)
Layer 2: 사용자 원국 (사주팔자 + 오행 분포 + 용신)
Layer 3: 캐릭터 톤 (감정/오행/십신 해석 프레임)
```

**3. RAG — 명리학 지식 기반**

프롬프트 하드코딩 대신 명리학 소스 텍스트를 벡터화. "AI가 지어낸 해석"이 아닌 "명리학 근거에 기반한 코칭". 지식 추가 시 재배포 불요.

**4. 만세력 라이브러리 사용**

LLM의 확률적 천문학 계산 대신 KASI 기반 라이브러리로 사주팔자를 정확하게 계산. AI는 해석만 담당.

## 사용한 AI 도구와 활용 방식

### Claude Code (개발 전 과정)
- **기획**: 시장 분석, 경쟁사 리서치(점신 MAU 95만, 포스텔러 매출 100억), 서비스 컨셉 설계
- **설계**: 시스템 아키텍처, API 설계, "선택형 상담 플로우" UX 설계를 코드 컨텍스트 기반으로 결정
- **구현**: 전체 코드베이스를 Claude Code가 프로젝트 구조를 이해한 상태에서 작성
- **검증**: TypeScript 타입 체크, 빌드 검증, 사주 계산 정확도 테스트

### Claude API (제품 핵심 기능)
- **tool_use 3종**: `deliver_coaching` (상담), `deliver_profile_summary` (프로필), `deliver_daily_action` (오늘의 한수)
- **캐릭터 프레임 분기**: 같은 사주 데이터를 다른 해석 렌즈(감정/오행/십신)로 읽는 3개 시스템 프롬프트
- **일일 행동 추천**: 오늘의 일진 + 사용자 원국을 분석하여 구체적 행동 1가지 추천

### 프롬프트 엔지니어링
- 명시적 부정 지시("조심하세요" 류 모호한 표현 금지)로 체감 정확도 향상
- action 필드에 "실행 가능한 구체적 행동"을 강제하는 규칙 설계
- "선택형 플로우"에서 수집된 구조화 맥락을 자연어 질문으로 변환하는 빌더 함수

### RAG (Retrieval-Augmented Generation)
- 명리학 소스 텍스트 → 청킹 → OpenAI 임베딩 → pgvector 저장
- 사용자 질문 + 사주 키워드로 관련 명리학 구절 검색 → Claude 프롬프트에 주입

```
│   └── api/
│       ├── saju/route.ts        # 사주 계산 API
│       ├── daily/route.ts       # 오늘의 한수 API
│       ├── profile/route.ts     # AI 프로필 생성
│       └── advice/route.ts      # 코칭 상담 API (RAG + Claude)
├── lib/
│   ├── saju.ts                  # 만세력 래퍼 (사주/오행/십신/용신)
│   ├── saju-data.ts             # 명리학 구조화 데이터
│   ├── prompts.ts               # 4계층 시스템 프롬프트
│   ├── claude.ts                # Claude API + tool_use (3종)
│   ├── rag.ts                   # RAG 파이프라인
│   ├── storage.ts               # localStorage 래퍼 + 일일 캐시
│   ├── suggestions.ts           # 사주 기반 질문 제안 로직
│   └── consult-flow.ts          # 선택형 상담 플로우 데이터
├── components/nav/              # Header + BottomNav + Footer
├── data/saju-knowledge/         # 명리학 소스 텍스트 (RAG 코퍼스)
├── scripts/embed-knowledge.ts   # 임베딩 파이프라인
└── supabase/migrations/         # DB 스키마
```

## 데모 시나리오

1. **온보딩**: 1990년 5월 15일 오후 2시 (양력) 입력
2. **홈 (오늘의 한수)**: "오늘 점심에 평소 말 안 하던 동료에게 먼저 커피를 제안해보세요" — 매일 다른 행동 카드
3. **상담 (선택형 플로우)**: 직장 → 이직 고민 → 이번 주 → (추가 맥락) → 친한 언니 캐릭터 선택 → 코칭 카드
4. **캐릭터 전환**: "다른 관점으로 보기" → 같은 맥락을 할머니 역술가로 재해석
5. **나 (프로필 + 기록)**: 사주팔자 + 오행 분포 + 상담 타임라인 + 피드백
