# 폴라리스 (Polaris) — AI 사주 라이프 코치

> 한 번 생년월일을 입력하면, 그 다음부터는 친구한테 고민 털어놓듯 사주를 봅니다.
> 5명의 코칭 캐릭터가 명리학 근거와 함께 구체적인 행동을 제안합니다.

## 실행 방법

### 1. 클론 및 설치

```bash
git clone https://github.com/jhlee95129/polaris.git
cd polaris
pnpm install
```

### 2. 환경변수

`.env.example`을 `.env.local`로 복사하고 API 키를 입력합니다.

```bash
cp .env.example .env.local
```

```
ANTHROPIC_API_KEY=           # Claude API (필수)
OPENAI_API_KEY=              # RAG 임베딩용 (필수)
NEXT_PUBLIC_SUPABASE_URL=    # Supabase 프로젝트 URL (필수)
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=  # Supabase 공개 키 (필수)
SUPABASE_SECRET_KEY=         # Supabase 서비스 키 (필수)
```

### 3. 데이터베이스 설정

Supabase 프로젝트에서 pgvector 확장을 활성화하고, `supabase/migrations/` 폴더의 마이그레이션을 순서대로 실행합니다.

```bash
# RAG 지식 임베딩 (1회만)
pnpm tsx scripts/embed-knowledge.ts
```

### 4. 개발 서버 실행

```bash
pnpm dev
```

`http://localhost:3000`에서 확인할 수 있습니다.

## 핵심 아키텍처

```
┌─────────────┐     ┌──────────────┐     ┌───────────────┐     ┌──────────────┐
│  고민 입력   │ ──→ │ 만세력 계산   │ ──→ │ RAG 검색      │ ──→ │ Claude       │
│  (자유 대화) │     │ (4기둥+대운)  │     │ (pgvector)    │     │ tool_use     │
└─────────────┘     └──────────────┘     └───────────────┘     └──────┬───────┘
                                                                      │
                                           ┌──────────────────────────┘
                                           ▼
                              ┌─────────────────────────┐
                              │  코칭 응답 (스트리밍)     │
                              │  + 사주 근거 (상세 분석)   │
                              │  + 추천 질문 (7개)        │
                              └─────────────────────────┘
```

**데이터 흐름**:
1. 사용자 메시지 수신 → 복채 차감
2. 사주 원국(4기둥 + 대운) + 대화 히스토리(최근 20턴) 로드
3. RAG 2-track 검색: (1) 대화 주제 기반 (2) 일간 특성 기반 → 최대 3개 chunk 추출
4. 캐릭터 페르소나 + 사주 컨텍스트 + RAG 지식으로 시스템 프롬프트 빌드
5. Claude Sonnet 4.6 스트리밍 응답 + `saju_basis` tool로 근거 구조화 추출
6. 응답 완료 시 AI가 맥락 기반 추천 질문 7개 자동 생성

## 주요 기능

### 대화형 사주 코칭 — 핵심 가설

> 사람들이 사주에서 진짜 원하는 건 "정해진 운명 보고서"가 아니라,
> **내 상황을 들어주고 사주 프레임으로 해석해주는 대화 상대**다.

- **자유 대화**: 카테고리 선택 없이 임의의 일상 상황을 던지면 사주 프레임으로 코칭
- **실시간 스트리밍**: SSE 기반 응답으로 즉각적인 대화 경험
- **사주 근거 투명성**: 매 응답마다 참조한 기둥·오행·추론·코칭을 "상세 분석 보기"로 열람 가능
- **세션 메모리**: 이전 대화를 기억하고 후속 코칭 제공 (*"지난번 이직 건은 어떻게 됐어?"*)
- **추천 질문**: AI가 대화 맥락에 맞는 후속 질문 7개 자동 제안

### 5명의 코칭 캐릭터

| 캐릭터 | 톤 | 설명 |
|--------|-----|------|
| 📜 선비 | 격식 있는 존댓말 | 조선 선비처럼 깊고 차분한 조언 |
| 🔮 무녀 | 직관적, 감각적 | 기운을 읽는 샤머니즘 상담가 |
| ⚔️ 장군 | 직설적, 전략적 | 핵심만 짚는 행동파 코치 |
| 🌸 선녀 | 부드럽고 따뜻한 | 자연의 비유로 위로하는 천상의 존재 |
| 👹 도깨비 | 장난스럽고 솔직 | 현대적 말투로 팩트를 던지는 장난꾸러기 |

동일한 명리학 지식 + RAG 파이프라인 위에 프롬프트 레이어만 분리하여 5가지 보이스를 구현합니다. 세션 중 캐릭터 전환 가능 — "같은 고민을 다른 시각에서" 들어볼 수 있습니다.

### 대시보드 — 매일 열어보는 이유

- **오늘의 폴라리스**: 매일 변하는 일일 운세 (코칭/주의사항/행운 숫자·색상/에너지 키워드)
- **8가지 일일 코칭 카드**: 오늘 운세, 직장, 연애, 재물, 인간관계, 행운, 주의, 내일 미리보기
- **추천 고민 토픽**: 4개 카테고리 × 16개 프리셋 질문으로 즉시 상담 시작
- **사주로 보는 나**: 성격 특성, 타고난 재능, 올해 흐름, 관계 패턴

### 사주 계산 — KASI 기반 정확도

- 4기둥(년/월/일/시주) + 오행 분포 + 십신 + 용신 계산
- 대운(10년 주기) 시퀀스 계산 및 마이페이지 시각화
- 음력/윤달 자동 변환, "시간 모름" 옵션 지원
- KASI(한국천문연구원) 데이터 기반, 진태양시 보정

### 복채 시스템 — 지속 가능한 비즈니스 모델

- 상담 1회 = 복채 1개 (직관적 단위)
- 가입 시 3개 무료 + 매일 출석 체크 +1
- 복채 부족 시 캐릭터 보이스로 안내 + 토픽 기억 → 충전 후 자동 전송
- 상점에서 패키지 구매 (현재 오픈 기념 무료)

### 기타 기능

- **대화 공유**: 공유 링크 → 카카오톡/SNS OG 프리뷰 지원
- **다크 모드**: 시스템/라이트/다크 테마 전환
- **반응형 UI**: 데스크톱(사이드바 + 채팅) / 모바일(풀스크린 채팅)
- **내 정보**: 닉네임·생년월일·캐릭터 변경, 사주 프로필 + 대운표
- **관리자 대시보드**: DAU, 리텐션, 퍼널 등 핵심 지표 (목업)

## 의사결정 기록

### 왜 채팅 인터페이스인가

핵심 가설이 **"사주 프레임으로 코칭하는 대화 상대"**입니다. 고민을 털어놓는 가장 자연스러운 인터페이스는 채팅입니다. 구조화된 콘텐츠 카드(플롯 방식)는 사전 설계된 범위 안에서만 작동하지만, 채팅은 "오늘 팀장이랑 또 부딪혔어" 같은 **임의의 일상 상황**에 대응할 수 있습니다.

3–5문장의 짧은 응답 + 되묻기 → 사용자가 다시 답하고 → 후속 코칭. 이 사이클이 리텐션의 핵심입니다. 추천 질문 7개 자동 생성은 "뭘 물어봐야 하지?"라는 마찰을 제거합니다.

### 왜 메모리가 핵심인가

신뢰는 누적에서 발생합니다. 1회성 리포트로는 "코치" 관계를 만들 수 없습니다. 멀티 세션 + 이전 대화 맥락 기반 follow-up 인사로, 재방문할 때마다 **"기억하고 있는 폴라리스"**를 경험합니다. 이것이 기존 사주 앱의 "설치 → 1회 사용 → 이탈" 패턴을 깨는 핵심 메커니즘입니다.

### 왜 5명의 캐릭터인가

처음에는 "단일 따뜻한 코치" 보이스로 시작했습니다. 그러나 사용자마다 선호하는 상담 톤이 다릅니다. Character.AI의 성공이 증명하듯, **"누가 말하느냐"가 "무엇을 말하느냐"만큼 중요**합니다. 동일한 명리학 지식과 RAG 파이프라인을 공유하되, 프롬프트 레이어에서 톤·말투·이모지 정책만 분리했습니다.

### 왜 플롯과 다른 길을 갔는가

플롯의 철학 — *"더 잘 맞히는 운세보다 더 도움이 되는 운세"* (제성원 본부장 인터뷰, 아주경제 2026.04) — 에 깊이 동의합니다. 폴라리스는 이 철학을 다른 구현 축에서 보완합니다.

플롯이 **"구조화된 콘텐츠 + 정령의 가이드"**로 사전 설계된 자기 이해 리포트를 제공한다면, 폴라리스는 **"임의의 일상 상황 + 누적 메모리 + 캐릭터 코치"**로 일상 코칭 상대의 자리를 채웁니다. 플롯이 의도적으로 비워둔 두 영역 — **임의 상황 대응**과 **누적 관계** — 를 폴라리스가 채우는 보완재 포지션입니다.

### 왜 RAG을 이렇게 설계했는가

**2-track 검색**: 대화 주제(예: "이직")로 관련 명리학 지식을 검색하는 동시에, 일간(예: "병화")으로 개인화된 해석 근거를 검색합니다. 두 트랙을 병합·중복 제거하여 최대 3개 chunk를 Claude에 주입합니다.

**데이터 거버넌스**: 명리학 도메인 텍스트는 외부 출처(명리학 도서 + 위키백과)에서 발췌·재구성했습니다. AI는 재구성 도구로만 사용하고 원문에 없는 해석 추가를 금지했습니다. Claude가 생성한 텍스트를 같은 Claude에게 grounding 데이터로 주입하면 circular 구조가 되어 RAG 의미가 사라지기 때문입니다.

**프롬프트 vs RAG 분리 기준**: 톤·코칭 원칙은 일관성이 중요하므로 시스템 프롬프트에 고정. 일간별 특성·지지 충합·격국 패턴 등 분량이 크고 사용자별로 다른 지식만 RAG으로 검색. *"무엇을 RAG으로 빼고 무엇을 프롬프트에 둘 것인가"의 판단*이 기술적 의사결정의 핵심이었습니다.

### 왜 tool_use로 사주 근거를 추출하는가

Claude의 응답은 자연어이므로, 사주 근거(참조한 기둥, 핵심 오행, 추론 과정)를 별도로 추출하지 않으면 사용자에게 구조화하여 보여줄 수 없습니다. `saju_basis` 도구를 정의하여 응답 생성과 동시에 구조화된 근거를 출력하게 했습니다.

이를 통해:
- 사용자는 "상세 분석 보기"로 AI의 추론 과정을 투명하게 열람
- 참조된 기둥별 한자·오행 색상을 시각화
- 코칭 주제를 자동 분류(career, love, finance 등)하여 추후 분석에 활용 가능

### 왜 만들지 않았는가

| 기능 | 이유 |
|------|------|
| 회원가입/소셜 로그인 | 닉네임 기반 간편 접근이 가설 검증에 충분 |
| 실결제 | 복채 구조 설계까지만. 실결제는 PMF 검증 후 |
| 운세 카테고리 탭 | "자유 대화"라는 가설과 정면 충돌 |
| 궁합 기능 | 1인 코칭이라는 핵심 가설에 집중 |
| 푸시 알림 | 비핵심. 매일 체크인 보상으로 재방문 유도 |

## AI 도구 활용

### Claude Code (개발 도구)

전체 코드베이스의 ~90%를 Claude Code로 작성했습니다. 작업 분해 방식:

1. **기획 단계**: SPEC.md에 가설, 페르소나, 기능 명세, 의사결정을 사전 문서화
2. **인프라 세팅**: Next.js + Supabase + shadcn 초기 구성
3. **코어 파이프라인**: 만세력 계산 → RAG → Claude 통합을 모듈 단위로 구현
4. **UI/UX**: 채팅, 대시보드, 온보딩을 반복 수정 (직접 피드백 → Claude Code 반영)
5. **캐릭터 시스템**: 5캐릭터 프롬프트 설계 + 톤 테스트로 반복 조정
6. **복채/비즈니스 로직**: DB 스키마 → API → 프론트 연동을 end-to-end로 구현
7. **검증**: 만세력 엣지케이스 5개 + 대운 계산 7개 = 22개 단위 테스트 작성/실행

Claude Code의 가치는 "코드를 대신 짠다"보다 **"의사결정을 빠르게 실험한다"**에 있었습니다. 프롬프트 톤을 바꾸고, 캐릭터를 추가하고, RAG 전략을 변경하는 것을 분 단위로 반복할 수 있었습니다.

### Claude Sonnet 4.6 (런타임 AI)

- **메인 코칭 엔진**: 사주 근거 + 캐릭터 보이스 + 구체적 코칭을 통합 생성
- **tool_use**: `saju_basis` 도구로 매 응답의 사주 근거를 구조화 추출
- **추천 질문**: 대화 맥락 기반 7개 후속 질문 생성
- **일일 운세**: 사주 원국 + 오늘 천기 기반 운세 생성
- **4계층 시스템 프롬프트**:
  - Layer 0: 캐릭터 페르소나 (5캐릭터 × 고유 톤·말투·이모지 정책)
  - Layer 1: 명리학 정적 지식 (오행 관계, 십신 분석, 코칭 원칙)
  - Layer 2: 사용자 원국 (4기둥, 일간, 현재 대운)
  - Layer 3: RAG 검색 결과 (2-track, 최대 3 chunk)

### OpenAI Embeddings (RAG)

- **모델**: `text-embedding-3-large` (2000 dimensions)
- **데이터**: 명리학 소스 마크다운 5개 파일 → 헤더 기준 청킹 → 벡터화
- **검색**: 대화 주제 + 일간 특성 2-track → 코사인 유사도 → Top-3 반환

### 만세력 라이브러리 검증

`@fullstackfamily/manseryeok`은 신생 라이브러리이므로 채택 전 5개 엣지케이스를 단위 테스트로 검증:

1. 입춘 전후 — 연주 경계 처리
2. 윤달 — 음력 변환 정확도
3. 야자시 — 일주 경계 처리
4. 시주 없음 — null 처리
5. 경계값 — 만세력 범위 한계

5개 모두 통과하여 채택했습니다.
